# Raw Findings: Cognitive Load Theory Applied to Executive Presentations

## Source 1: Paul & Seniuk Cicek (2021) — "The Cognitive Science of PowerPoint"
- **Publication:** Proceedings of CEEA, University of Manitoba
- **DOI:** https://doi.org/10.24908/pceea.vi0.14872

### Key Findings:
- Many slides are poorly designed with HIGH cognitive load
- They either contain too much information OR have poorly organized information
- **Inverse correlation between cognitive overload and memory** — as cognitive load increases, retention decreases
- Reducing cognitive load leads to more effective presentations: improved communication, retention, and instruction
- Covers: cognitive load theory, expert-novice divide, dual-channel theory, gestalt principles, constructivism
- Practical application demonstrated with authentic classroom slide examples

## Source 2: Garr Reynolds — Presentation Zen Design Principles
- **Publication:** garrreynolds.com/design-tips

### 14 Design Principles (with cognitive load implications):

1. **Keep it simple** — Nothing superfluous. Plenty of empty/white space. Edward Tufte: "What can be done with fewer is done in vain with more."

2. **Make it visual** — Picture Superiority Effect: pictures remembered better than words, especially after 30+ seconds. "Use pictures and words together, reinforcing the same information." (Lidwell, Holden, Butler — Universal Principles of Design)

3. **Don't create "slideuments"** — Slides ≠ documents. Best slides are meaningless without the speaker. Attempting to make slides serve as both creates the worst of both worlds.

4. **High Signal-to-Noise Ratio** — "The ratio of relevant to irrelevant information in a display." "Every element in a design should be expressed to the extent necessary, but not beyond." Remove unnecessary lines, footers, logos, decorative elements.

5. **Design for the back of the room** — Nancy Duarte: "Presentations are a 'glance media' — more closely related to billboards than other media. Ask yourself whether your message can be processed effectively within three seconds."

6. **Make images large, often fullscreen** — One large, powerful image > several smaller ones.

7. **Use contrast** — Strong focal point. "We are all wired to notice differences." Weak contrast makes visuals boring AND confusing.

8. **Rule of Thirds** — Place key elements at intersection points for natural visual balance.

9. **Clear charts & graphs** — Three sub-principles:
   - **Restrain:** Don't add more than necessary
   - **Reduce:** Remove nonessential elements
   - **Emphasize:** Use color/contrast to highlight what matters
   - Use **declarative statements** as chart titles, not descriptive labels

10. **Choose fonts well** — Sans serif generally preferred for projected slides. Make them LARGE.

11. **Use short video clips** — Change of pace, increases interest, promotes active processing.

12. **Use colors with purpose** — Color improves learning comprehension and retention. Cool colors for backgrounds (recede), warm for foreground elements (advance). Few colors, consistent throughout.

13. **Limit animation/transitions** — Movement is what the brain notices most. Subtle > flashy. Too much = distracting.

14. **Spend time in slide sorter** — Segmenting Principle: people comprehend better when info is in small chunks. Get the 35,000-ft view.

## Source 3: Mayer's Cognitive Theory of Multimedia Learning (CTML)
- **Referenced in:** Wolfe et al. (2023), SlideModel McKinsey analysis, Paul & Seniuk Cicek (2021)

### Core Principles:
| Principle | Description | Slide Application |
|-----------|-------------|-------------------|
| Coherence | Remove extraneous material | Every element must serve the message |
| Signaling | Use cues to highlight organization | Bold, color, icons to guide the eye |
| Redundancy | Don't duplicate visual + narration text | If you'll say it, don't write it on screen |
| Spatial Contiguity | Place related items near each other | Labels near graphics, not in separate legends |
| Temporal Contiguity | Present related items simultaneously | Show graphic and narration together |
| Segmenting | Break into smaller chunks | Multiple simple slides > one complex slide |
| Pre-training | Introduce key concepts first | Define terms before diving into architecture |
| Modality | Use visual + audio channels | Narration + visual > on-screen text + visual |
| Multimedia | Use words + pictures together | Text + images > text alone |
| Individual Differences | Effects stronger for novices | Executive audiences may vary in technical expertise |

## Source 4: Sweller — Cognitive Load Theory Types
| Load Type | Definition | Design Strategy |
|-----------|-----------|----------------|
| Intrinsic | Inherent complexity of material | Simplify; use analogies; break into components |
| Extraneous | Load from poor design | Minimize clutter, noise, unnecessary decoration |
| Germane | Productive load for schema building | Use visual schemas, logical groupings, patterns |

**Key insight for executive presentations:** Executives already have HIGH intrinsic load (processing many decisions daily). Extraneous load from poor slide design compounds this. Minimize extraneous load to preserve cognitive capacity for the actual decision.

## Source 5: Miller's Law — Working Memory Limits
- **Citation:** Miller, G.A. (1956). "The Magical Number Seven, Plus or Minus Two"
- **Application:** Working memory holds 3-4 items reliably (updated estimate; original was 7±2)
- **Implication:** Maximum 3 supporting points per argument (Hazeldine, 2026)
- **Implication:** Maximum 3-4 key metrics per dashboard slide
