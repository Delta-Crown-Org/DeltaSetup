# Raw findings: W3C/WAI WCAG and evaluation tools

## WCAG 2 Overview

Source: https://www.w3.org/WAI/standards-guidelines/wcag/

Extracted findings:

- WCAG documents explain how to make web content more accessible to people with disabilities.
- Web content includes natural information (text, images, sounds) and code/markup that defines structure and presentation.
- WCAG 2.2 has 13 guidelines organized under four principles: perceivable, operable, understandable, and robust.
- Success criteria are at levels A, AA, and AAA and determine conformance.
- WCAG 2.2 was published on 2023-10-05 and updated on 2024-12-12.
- W3C encourages use of the latest WCAG version.
- Content that conforms to WCAG 2.2 also conforms to WCAG 2.1 and 2.0, aside from the 4.1.1 Parsing obsolete nuance.

## Selecting Web Accessibility Evaluation Tools

Source: https://www.w3.org/WAI/test-evaluate/tools/selecting/

Extracted findings:

- Evaluation tools help quickly identify potential accessibility issues.
- Tools can provide fully automated checks and help with manual review.
- Tools cannot check all accessibility aspects automatically; human judgment is required.
- Tools can produce false or misleading results.
- Evaluation tools cannot determine accessibility; they can only assist in doing so.
- Different tools support different roles, standards, file types, scopes, operating systems, and reporting modes.
