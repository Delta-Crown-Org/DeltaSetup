# Baymard Institute Mobile UX Research Summary

**Source**: Baymard Institute - E-commerce UX Research
**URL**: https://baymard.com/blog/collections/mobile
**Date Retrieved**: March 31, 2025
**Source Tier**: Tier 1 (Highest - Large-scale Usability Research)

## Research Scope

- **200,000+ hours** of UX research
- **700+ UX Best Practice Guidelines**
- **18,000+ Structured Design Examples**

- **40,000+ implementation examples** from 150+ leading ecommerce sites

## Key Mobile UX Articles and Findings

### Navigation Performance

**"Homepage and Category Navigation UX 2025"**
- **67% of mobile sites** have mediocre-to-poor navigation performance
- Key issues in mobile navigation structure and findability

**"Make Product Categories the Top-Level Navigation Items on Mobile Sites"**
- **33% of sites** make navigation overly difficult
- Mobile users rely on main navigation to understand product catalog
- Recommendation: Keep category structure flat and visible

### Touch Targets and Interaction

**"3 Strategies for Handling Accidental 'Taps' on Touch Devices"**
- Common issue: users accidentally tap items on mobile pages
- Strategies needed to handle inadvertent clicks
- Critical for preventing frustration and errors

**"Hit Areas in Visual Elements"**
- **33% of sites** don't make clear where hit areas lead
- Causes product finding slowdown and user disorientation

### Form Usability

**"Never Use Inline Labels" (2013, foundational)**
- Inline labels cause mobile form usability issues
- Recommendation: Place labels above fields

**"Avoid Splitting Single Input Entities"**
- Split fields (like phone numbers, credit cards) cause input errors
- Keep related inputs together

**"Touch Keyboard Implementations"**
- **60% of sites** still get keyboard basics wrong (2015-2023)
- Only **9% improvement** since 2013
- Critical for mobile data entry

### Product Page Design

**"Always Use Thumbnails to Represent Additional Product Images"**
- **76% of mobile sites don't** use image thumbnails
- Product images are key to decision-making
- Thumbnails ubiquitous on desktop but rare on mobile

**"6 Important Aspects of Well-Performing Mobile Product Page Breadcrumbs"**
- **36% of sites** don't provide full category paths
- Mobile users rely on breadcrumbs for navigation and orientation

**"Avoid Using Subpages within the Product Details Page"**
- **26% of sites** use subpages for mobile content
- Leads to users overlooking content entirely
- Severe issue in mobile product pages

### Mobile Search

**"Always Provide a Submit Button Adjacent to the Search Field on Mobile"**
- **21% don't** provide explicit submit button
- Causes significant friction for users

**"Offer Relevant Autocomplete Suggestions for Closely Misspelled Search Terms"**
- **69% don't** support misspellings in autocomplete
- Users at risk of misunderstanding product catalog

**"Always Persist Users' Search Queries"**
- **37% don't** persist search queries
- Slows down product finding significantly

### Mobile Checkout and Forms

**"4 Design Patterns That Violate 'Back' Button UX Expectations"**
- **59% of sites** get back button behavior wrong
- Users expect back button to take them to perceived previous page

**"Checkout UX: Avoid 'Apply' Buttons for Most Fields"**
- **22% of sites** still use apply buttons
- Users overlook or misunderstand them
- Causes data loss and abandonment

**"The 'Credit Card Number' Field Must Allow and Auto-Format Spaces"**
- **80% don't** auto-format credit card spaces
- Makes input significantly harder on mobile

**"Phone Number UX: Always Explain Why the 'Phone Field' Is Required"**
- **39% don't** explain phone field requirement
- **14% of users** will abandon checkout

### Mobile Performance and Loading

**"Product List UX: The Number of Products to Load by Default"**
- **52% get it wrong**
- Optimal number varies by product type and mobile vs desktop

**"Display 'Applied Filters' in an Overview"**
- **32% aren't** using best practices for filtering
- Overviews speed up product finding and help orient users

### Mobile Layout Issues

**"42% of Mobile Homepages Risk Setting Wrong Expectations for Their Users"**
- 70% of mobile users scroll homepage to "get a feel for the site"
- Homepage design causes misinterpretation of product catalog

**"Mobile Web: Scale Product Images Proportionally in Mobile Landscape Mode"**
- **52% of sites don't** scale images in landscape
- Users expect images to scale when switching orientation

**"Return Users to the Same Place in the Product List When Returning from the Product Page"**
- **13% don't** return users to correct position
- Slows product finding considerably

**"Understanding Mobile E-Commerce UX: 5 Overarching Issues" (2021)**
- 5 core issues shared by all mobile sites
- Any of these can lead to user abandonments

## Mobile App vs Mobile Web

**Key Insight**: 98% of product-finding guidelines for mobile websites also apply to native mobile apps.

## Implications for Franchise Portals

### Critical Issues to Address

1. **Navigation Clarity**
   - Flat, visible category structure
   - Clear top-level navigation
   - Full breadcrumb paths

2. **Touch Target Optimization**
   - Clear hit areas
   - Prevent accidental taps
   - Proper spacing between elements

3. **Form Design**
   - Labels above fields (not inline)
   - Auto-formatting for complex inputs
   - Minimize field splitting

4. **Search Experience**
   - Visible submit buttons
   - Autocomplete with misspelling support
   - Persist queries across sessions

5. **Image Handling**
   - Thumbnail galleries for documents/resources
   - Scale properly in landscape
   - Support pinch/zoom gestures

6. **Back Button Behavior**
   - Meet user expectations
   - Return to perceived previous page
   - Maintain state properly

7. **Content Organization**
   - Avoid subpages for mobile content
   - Progressive disclosure instead
   - Keep content on single page when possible

### Performance Benchmarks

- 67% of mobile sites have navigation issues
- 78% of mobile sites have product list issues
- Average mobile checkout abandonment: 22% due to complexity
- Most common issues: touch targets, navigation, forms

### Recommendations for Franchise Portals

1. **Prioritize Core Mobile UX**
   - Fix navigation structure first (impacts 67% of sites)
   - Optimize touch targets and hit areas
   - Streamline form interactions

2. **Simplify Information Architecture**
   - Flat category structure for mobile
   - Clear primary navigation items
   - Consistent breadcrumb implementation

3. **Optimize for Touch Input**
   - Minimum 48dp touch targets (Material Design)
   - Clear visual feedback
   - Prevent accidental activations

4. **Streamline Common Tasks**
   - Resource access
   - Training completion
   - News consumption
   - Quick lookups

5. **Test on Real Devices**
   - Multiple screen sizes
   - Portrait and landscape orientations
   - Various network conditions
   - Different input methods (touch, stylus)
